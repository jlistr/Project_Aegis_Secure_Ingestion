## ?? Best Practices

### 1. **Start Broad, Then Refine**
Begin with broad prompts to explore ideas, then narrow down with specific follow-ups.

### 2. **Reference Previous Outputs**
Always reference earlier agent outputs in subsequent prompts to maintain context.

### 3. **Organize Your Documentation**
```
/workspace/docs/
??? planning/
?   ??? roadmap.md
?   ??? mvp-definition.md
?   ??? resource-plan.md
??? research/
?   ??? competitive-analysis.md
?   ??? market-sizing.md
?   ??? pricing-research.md
??? architecture/
?   ??? system-design.md
?   ??? api-specs/
?   ??? database-schema.md
??? sales/
?   ??? sales-playbook.md
?   ??? pricing-strategy.md
?   ??? email-templates.md
??? proposals/
    ??? investor-pitch.md
    ??? client-proposals/
```

### 4. **Iterate and Validate**
- Use agent outputs as starting points
- Review and refine with human expertise
- Validate assumptions with research
- Test architectural decisions with POCs

### 5. **Chain Agents Effectively**
- Save outputs from each agent
- Reference saved outputs in next prompts
- Build comprehensive documentation progressively
- Maintain context across interactions
