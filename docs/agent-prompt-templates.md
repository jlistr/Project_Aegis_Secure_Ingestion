# Cursor Cloud Agent Prompt Templates

Ready-to-use prompt templates for different SaaS startup phases.

---

## Planning & Strategy Templates

### MVP Definition Template
```
I'm building a SaaS platform for [DOMAIN/PROBLEM]. Help me define the MVP by:

1. Identifying the core problem we're solving
2. Listing essential features that solve this problem (must-have only)
3. Prioritizing features by:
   - User value (high/medium/low)
   - Technical complexity (high/medium/low)
   - Development time (weeks/months)
4. Defining success metrics for MVP launch
5. Identifying what we can defer to post-MVP

Consider lean startup principles and time-to-market constraints.
Output: Prioritized feature list with justifications.
```

### Roadmap Creation Template
```
Create a 12-month product roadmap for [SAAS_NAME]:

1. **Q1 (Months 1-3)**: MVP development
   - Features to include
   - Technical milestones
   - User validation goals

2. **Q2 (Months 4-6)**: Launch and iteration
   - Launch features
   - Post-launch improvements based on feedback
   - Marketing milestones

3. **Q3 (Months 7-9)**: Growth phase
   - New features based on user demand
   - Scale infrastructure
   - Partnership opportunities

4. **Q4 (Months 10-12)**: Optimization and expansion
   - Feature refinements
   - Market expansion
   - Revenue milestones

For each quarter, include: deliverables, dependencies, resources needed, success metrics.
```

## Research & Market Analysis Templates

### Competitive Analysis Template
```
Research and analyze competitors for [SAAS_NAME] in the [DOMAIN] space:

1. **Direct Competitors** (solve same problem):
   - Product names and brief descriptions
   - Pricing models
   - Key strengths and weaknesses
   - Market positioning

2. **Indirect Competitors** (alternative solutions):
   - What alternatives exist
   - Why users might choose them
   - How we differentiate

3. **Competitive Matrix**:
   Create a feature comparison table with:
   - Core features
   - Pricing tiers
   - Target market
   - Technology stack (if public)

4. **Market Gaps**:
   - What do competitors miss?
   - What pain points aren't addressed?
   - Opportunities for differentiation

Output: Structured comparison document with actionable insights.
```

### Market Sizing Template
```
Estimate market size for [SAAS_NAME] targeting [TARGET_MARKET]:

1. **Total Addressable Market (TAM)**:
   - Total market revenue opportunity
   - Assumptions and methodology
   - Data sources

2. **Serviceable Addressable Market (SAM)**:
   - Subset of TAM we can realistically serve
   - Geographic limitations
   - Market segment constraints

3. **Serviceable Obtainable Market (SOM)**:
   - Realistic market share we can capture in 3-5 years
   - Based on competitive landscape
   - Growth projections

4. **Market Segmentation**:
   - Primary segments
   - Segment sizes
   - Segment-specific pricing potential

Output: Market analysis with calculations and assumptions documented.
```

### Pricing Research Template
```
Research pricing models and pricing benchmarks for [SAAS_NAME] in [INDUSTRY]:

1. **Competitor Pricing Analysis**:
   - Pricing tiers and structure
   - Feature differentiation per tier
   - Add-on pricing
   - Volume/enterprise pricing

2. **Industry Benchmarks**:
   - Average pricing for similar SaaS products
   - Revenue per user (ARPU) benchmarks
   - Pricing as % of customer revenue

3. **Value-Based Pricing Considerations**:
   - Cost savings provided to customers
   - Revenue increase potential
   - Time saved value
   - Risk mitigation value

4. **Recommended Pricing Strategy**:
   - Suggested pricing tiers
   - Justification for each tier
   - Discount strategy
   - Annual vs monthly pricing

Output: Pricing research report with recommendations.
```

## Design & Architecture Templates

### System Architecture Template
```
Design the technical architecture for [SAAS_NAME] with the following requirements:

**Requirements:**
- [List key requirements: e.g., multi-tenant, real-time, scalability to 10K users, etc.]
- [Technology preferences: e.g., AWS, Node.js, PostgreSQL, etc.]
- [Integration needs: e.g., payment processing, email service, etc.]

**Architecture Should Include:**
1. **High-Level Architecture Diagram**:
   - Components and services
   - Data flow
   - External integrations

2. **Technology Stack**:
   - Frontend: framework, libraries, build tools
   - Backend: runtime, frameworks, API design
   - Database: type, replication strategy
   - Infrastructure: cloud provider, services used

3. **Scalability Considerations**:
   - Horizontal vs vertical scaling
   - Load balancing strategy
   - Caching strategy
   - Database scaling approach

4. **Security Architecture**:
   - Authentication/authorization
   - Data encryption (at rest, in transit)
   - Security best practices
   - Compliance considerations

5. **Deployment Architecture**:
   - CI/CD pipeline
   - Environment strategy (dev/staging/prod)
   - Monitoring and logging
   - Disaster recovery

Output: Architecture document with diagrams and detailed explanations.
```

### API Design Template
```
Design a RESTful API for [SAAS_NAME] with the following features:

**Core Resources:**
- [List main resources: e.g., Users, Projects, Tasks, etc.]

**API Requirements:**
1. **Authentication**: [JWT/OAuth/API Keys]
2. **Rate Limiting**: [Requests per minute]
3. **Versioning**: [Strategy: URL/header]
4. **Error Handling**: [Standard error format]
5. **Pagination**: [Cursor/page-based]

**For Each Resource, Specify:**
- Endpoints (GET, POST, PUT, DELETE, PATCH)
- Request/response schemas (JSON)
- Query parameters
- Status codes
- Example requests/responses

**Additional Considerations:**
- Webhooks (events to subscribe to)
- Batch operations
- Filtering and sorting
- Field selection

Output: Complete API specification document (OpenAPI/Swagger format preferred).
```

### Database Schema Template
```
Design the database schema for [SAAS_NAME]:

**Requirements:**
- Multi-tenancy: [Database/schema/row-level isolation]
- Scale: [Expected data volume]
- Performance: [Key query patterns]

**Schema Should Include:**
1. **Entity Relationship Diagram (ERD)**:
   - All tables
   - Relationships (one-to-one, one-to-many, many-to-many)
   - Foreign keys

2. **Table Definitions**:
   For each table:
   - Columns with data types
   - Primary keys
   - Foreign keys
   - Indexes (with justification)
   - Constraints

3. **Data Access Patterns**:
   - Common queries
   - Join patterns
   - Read vs write patterns

4. **Migration Strategy**:
   - Initial schema
   - Future changes approach
   - Data migration considerations

Output: Database schema documentation with ERD and migration scripts.
```

## Usage Instructions

1. **Copy a template** that matches your need
2. **Replace placeholders** in brackets [LIKE_THIS] with your specific details
3. **Paste into Cursor Cloud Agent** and execute
4. **Review and refine** the output
5. **Iterate** with follow-up prompts based on results

---

## Tips for Best Results

1. **Be Specific**: Include details about your domain, target market, and constraints
2. **Provide Context**: Reference your existing documentation or decisions
3. **Ask for Formats**: Specify output format (markdown, JSON, diagrams, etc.)
4. **Iterate**: Use agent outputs as starting points, then refine
5. **Combine Templates**: Chain multiple templates for comprehensive results
